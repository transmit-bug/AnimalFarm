# UI/UX 方向

> 玩家如何与游戏交互。框架级定义，具体执行细节待后续确定。

---

## 信息呈现

**框架式，深度融合美术。**

有明确的 UI 框架（状态栏、手牌区、事件区），但框架本身用美术风格包装——像旧文件的边框、像农场的木板、像泛黄的纸张。

**原则：UI 不是"游戏界面贴着美术"，而是"界面就是美术"。**

UI 本身也是叙事的一部分：
- 七诫被篡改时，UI 上的文字跟着变
- 审查发生时，UI 本身传达压迫感
- 资源状态恶化时，UI 的视觉质感跟着变化（纸张变皱、墨迹变淡）

## 卡牌交互

**混合式，交互模式由卡牌语义决定。**

| 卡牌类型 | 交互方式 | 原因 |
|---|---|---|
| 每日行动卡（劳动/社交/调查） | 点击即出 | 不需要目标，触发事件场景 |
| 思维卡 | 点击即出 | 改变决策方式，无目标 |
| 意识形态卡 | 点击即出/被动生效 | 持续改变事件解读 |
| 关系卡 | 点击选中 → 选择目标 | 需要指定对哪个动物使用 |
| 情报卡（部分） | 点击选中 → 选择目标 | 对某个NPC使用来解锁信息 |
| 情报卡（部分） | 被动生效 | 拼凑真相，无需目标 |

**关键原则：**
- 交互模式由卡牌自身决定，不是系统强制切换
- 需要目标的卡有视觉提示（如小箭头图标），玩家一看就知道
- 日常操作保持简洁，关键事件时交互可以加重（全屏面板、时间压力等）

## NPC 交互

**混合式，交互形态服务于叙事场景。**

| 场景类型 | 交互形态 | 说明 |
|---|---|---|
| 日常社交 | 对话树 + 手牌选项并列 | NPC立绘 + 对话文本 + 底部手牌区，获取卡选项有视觉区分 |
| 审查事件 | 全屏面板 | 紧张感拉满，选项有限，可能有时间压力 |
| 秘密对话 | 低语式文本 | 选项隐蔽，特殊字体/颜色，营造私密感 |
| 内心独白 | 思维卡牌 + 文字 | 没有NPC，只有内心声音，类似《极乐迪斯科》技能对话 |
| 调查/探索 | 场景面板 | 点击场景中的元素触发不同内容 |
| 特殊事件 | 定制交互 | 如"篡改七诫"变成可擦写的墙、"秘密集会"变成暗处剪影 |

**关键原则：**
- 所有形态共享同一套视觉语言（纸质载体、版画风格、框架式UI），保证一致性
- 交互形态服务于叙事场景，不是系统统一
- 不排除后续出现全新的交互形态

---

## 设计原则

- **UI 即叙事** — 界面不是功能性的壳，而是故事的一部分
- **交互跟随语义** — 怎么交互取决于这张卡/这个场景"是什么"，不是"系统要求"
- **一致性来自视觉语言** — 交互形态可以多样，但视觉语言统一
- **保持简洁** — 日常操作不繁琐，复杂交互只在关键时刻出现
